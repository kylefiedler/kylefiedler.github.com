Infura Interface Overhaul & Design System refactor

Infura

2022–23

  • Design Systems
  • Systems Thinking
  • Visual Design
  • Cross-functional Collaboration
  • HTML/CSS

The Problem

Infura’s design had fallen behind user expectations—and competitors. We regularly heard feedback from users that the UI felt disjointed, dated, and inconsistent. Years of fast-paced feature work had created a tangled component system. There were over 10 types of buttons used across the product, many of them one-offs. Components were inconsistently implemented in both Figma and code.

The result:

  • Design and engineering were duplicating work
  • Shipping new features took longer than necessary
  • The overall experience lacked the polish users expected from a developer tool of our scale

Getting Started

I kicked off the project by reviewing feedback already collected by our support and product teams. Using this, I wrote a proposal outlining key problems and a plan to tackle them, with clear, tangible success metrics:

  • Reduce time from design to production
  • Increase design consistency and reduce the number of components to maintain
  • Improve user satisfaction and retention—eliminate UI as a reason for churn

Visual Exploration & Stakeholder Alignment

I began by exploring updated visual styles for Infura’s core app screens, using recent brand design work as a starting point. After refining with feedback from the brand team and design leadership, I opened the work to broader input from the design and frontend teams.

Once we had early alignment, I broke the exploration down into core components we knew the app needed. I took inventory of the full interface and mapped components to real usage across the product. I organized this work using Atomic Design principles, to help guide structure and communication.

Rebuilding the System, One Component at a Time

For each component, I followed a full design-to-implementation lifecycle:

  • Update the design in Figma
  • Implement in React Storybook
  • Integrate into the app

Each step informed the others. As new edge cases emerged during integration, I revisited and refined components in Figma and Storybook. Since I was also managing the project, I had visibility into where gaps were forming—and could quickly resolve them across design and engineering.

Shipping in Iteration, Not All at Once

Rather than waiting for a big relaunch, we shipped components as they were completed. This meant users experienced a gradual improvement over time—never a jarring change—and gave the team time to adapt to new patterns organically.

We also used each rollout as a chance to fix small UI/UX bugs that had accumulated. This not only improved the experience but clarified which components should be used where, reducing future inconsistencies.

Results & Impact

The design system overhaul led to significant improvements across design, engineering, and user satisfaction:

  • Faster time from design to production for new UI features
  • Reduced component count by over 56%, improving maintainability
  • Fewer UI bugs, more than 53 tickets cleared, and clearer component usage across the codebase
  • Zero user complaints about design during post-rollout churn analysis
  • Improved collaboration between design and engineering through shared language and documentation

This wasn’t just a visual refresh, it rebuilt the foundation for how we shipped product at Infura. The end result: a cleaner, more unified user experience that scaled with our team and our users.
