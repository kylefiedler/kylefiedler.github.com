Today was a first day for one of the designers on the project that I have been on for a while. He is new to thoughtbot and this was the first time that he had worked with another designer at thoughtbot. We went over our pull request process and after he had one he was grateful. Something that I’ve felt annoying before and the long way around he saw the good in.

With each pull request is a chance to learn something new about the code that you have written. That second look, fresh eyes and possibly more knowledgeable might make you and your code better. After seeing that, I wondered if there was a way for us to do this with user experience. Besides usability test is there another way to get fresh eyes on our work?
